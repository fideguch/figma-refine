# figma-refine

> [日本語版](README.md)

A Claude Code skill for context-aware Figma design refinement. Understands your project's background (brand, target users, competitors, domain terminology) and refines Figma designs accordingly. Provides an asynchronous design review workflow through ReviewNote components.

## Overview

- Loads project context from `designs/` folder (builds a "Design Personality")
- Detects ReviewNote components on Figma canvas and interprets modification intent
- Applies modifications via Figma MCP with project-specific awareness
- 3-layer screenshot verification (Component → Frame → Page) × dual-axis quality judgment (design perspective + user perspective)

## Key Features

**Context-driven**: Instead of generic "AI-like UI", makes design decisions based on your project's brand colors, mood keywords, competitive references, and target user demographics.

**3-layer × dual-axis verification**: Captures screenshots at component, frame, and page levels after each modification, then judges quality from both design perspective (token compliance, accessibility) and user perspective (confusion risk, naturalness, brand consistency).

## Installation

git clone https://github.com/fideguch/figma-refine.git ~/.claude/skills/figma-refine

## Prerequisites

- **Figma MCP**: Figma MCP server connected to Claude Code
- **Figma file**: URL of the target Figma Design file
- **designs/ folder** (recommended): Project requirements generated by `/requirements_designer`

## Usage

```
/figma-refine
```

Trigger words: "figma-refine", "design refinement", "review note", "figma review"

## Workflow

```
Step 1: Context Load   — Read designs/ and build Design Personality
Step 2: Bootstrap      — Create ReviewNote master component (first run only)
Step 3: Process        — Detect unread ReviewNotes → apply modifications
Step 4: Verify         — 3-layer × dual-axis screenshot verification
```

## Supported Modifications

| Code | Category | Examples |
|------|----------|---------|
| M-001 | Color | Fill/stroke color change |
| M-002 | Text | Content, font size, font weight |
| M-003 | Size | Width/height (non-auto-layout only) |
| M-004 | Corner | Corner radius |
| M-005 | Spacing | Padding, gap (auto-layout frames) |
| M-006 | Visibility | Show/hide |
| M-007 | Opacity | 0.0 to 1.0 |

## Limitations

- Node creation/deletion not supported (structural changes require manual intervention)
- Image replacement requires image hash (not available via ReviewNote)
- Variant switching requires component set understanding
- Effects (shadow, blur) have complex parameters requiring visual confirmation
